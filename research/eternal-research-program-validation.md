# Validation — Is This an "Eternal" (Open-Ended) Research Program?

*Assessed with Lakatos's framework for scientific research programmes. Verdict
governs whether the program may legitimately call itself perpetual/self-renewing.
Read with [`vision-and-scope.md`](./vision-and-scope.md).*

> **Definition adopted.** "Eternal" = **open-ended and self-renewing** (like medicine
> or ecology — never "finished" because the subject keeps renewing), **not**
> "infinite / beyond refutation." The latter is not a research program; it is the
> degenerating failure mode this document exists to prevent.

---

## 1. The Lakatosian structure (does it have one?)

A durable programme needs a stable **hard core**, a revisable **protective belt**,
and a generative **positive heuristic**. This program has all three.

### Hard core (defended, not tested directly)
> *Adaptive capability has a domain-invariant structure that can be measured and
> that predicts how systems survive, learn, and evolve.*

### Protective belt (revisable without abandoning the core)
- The specific 11 levels and their boundaries
- The rubric (v1.1) and survey instrument
- The chosen formal invariant (FEP / Generalized Darwinism / …)
- The particular domains and outcome measures

### Positive heuristic (the engine of open-endedness)
Each answered question *generates* the next — this is what makes it perpetual:
```
Does the structure transfer?  → What mechanism produces it?
What mechanism?               → Which formal invariant captures it?
Which invariant?              → Does it hold in new substrates (AI, ecosystems)?
New substrates?               → How does it scale to civilizations / 1,000-yr design?
At civilization scale?        → New domains, new shocks, new technologies … ↺
```
Because adaptation's subject matter renews itself, the heuristic does not terminate.
**This is the legitimate basis for calling the program open-ended.**

## 3. The decisive test: progressive vs. degenerating

Lakatos's rule: a programme is **progressive** while it predicts *new* facts, and
**degenerating** once it only adds patches to protect the core after the fact.

- **Progressive evidence (must accumulate):** novel, pre-registered predictions that
  come true — e.g., maturity predicts shock-recovery in a domain *before* it was tested.
- **Degenerating warning signs (must be policed):** new "levels" or "tiers" added only
  to absorb anomalies; grander vocabulary without new predictions; unfalsifiable scope
  expansion. **This is precisely what the source series did from v22→v41** — each
  version added a metaphysical tier but **no new testable prediction.** That is a
  textbook degenerating programme, and the cautionary example this program must not repeat.

## 4. Conditions for legitimate "eternal" status

The program may call itself open-ended/perpetual **only while all hold**:

- [ ] The hard core stays **fixed**; revisions happen in the belt.
- [ ] Each cycle yields **≥1 novel, pre-registered prediction** (progressiveness).
- [ ] Anomalies trigger **belt revision or core abandonment** — never ad-hoc tiers.
- [ ] Scope growth is **earned by evidence**, not asserted (the scope contract).
- [ ] **Renewal infrastructure** exists: open data/instruments, successors, replication.

If these lapse, "eternal" degrades to "degenerating" and the honest move is to **fork
or retire** the core — not defend it forever.

## 5. Verdict

**Validated — conditionally — as an open-ended *progressive* research programme.**

- ✅ It has a clean Lakatosian structure and a genuinely non-terminating heuristic,
  so "self-renewing/perpetual" is **earned in principle**.
- ⚠️ It is **not yet progressive in fact** (no novel confirmed predictions) and has
  **no renewal infrastructure** — so the "eternal" label is *prospective*, not current.
- ❌ "Eternal" in the metaphysical/infinite sense is **rejected**, and the v22→v41
  pattern is logged as the degeneration it must avoid.

> **Bottom line:** It can become an enduring, multi-generational science of adaptation
> — *if and only if* every cycle keeps paying the progressiveness tax. Perpetuity is
> not a property you declare; it is a status you re-earn each cycle by predicting
> something new and being willing to be wrong.
