# Research Questions

*Canonical statement of the thesis's research questions. One primary question (RQ1)
decomposed into four supporting questions (RQ2–RQ5), each mapped to a falsifiable
proposition and the literature gap it closes. This is the reference list of record;
where [`doctoral-proposal.md`](./doctoral-proposal.md) §4 and
[`../02-theoretical-framework/theoretical-framework.md`](../02-theoretical-framework/theoretical-framework.md)
§6 differ, they defer to this document.*

---

## Central question

> **Can adaptive capability be represented as a domain-invariant hierarchical
> construct that predicts adaptive outcomes across multiple domains — without
> per-domain modification?**

This is the empty cell of the maturity-model landscape: *cross-domain + empirically
validated + predictive*. RQ1 asserts that cell can be filled; RQ2–RQ5 are the four
independent walls that must each hold for the assertion to survive. See
[`research-gaps.md`](./research-gaps.md) for the gap each question closes.

---

## RQ1 — Primary

**Can adaptive capability be represented as a domain-invariant hierarchical construct
that predicts adaptive outcomes across multiple domains?**

The unifying question. It is true **only if all four supporting questions resolve
favorably** — a conjunction that makes the thesis falsifiable rather than a
self-confirming narrative. The construct under test is the 11-level ladder
(Reality → Observation → Data → Knowledge → Intelligence → Wisdom → Purpose →
Action → Learning → Adaptation → Evolution).

## RQ2 — Measurement

**Can adaptive capability be measured reliably and validly?**

Tests whether the construct is operationalizable: objective per-level indicators,
inter-rater reliability (Cohen's κ / Krippendorff's α ≥ 0.7), and construct validity
(the 11 levels do not collapse into fewer latent factors). Prerequisite for every
later question — an unmeasurable construct makes RQ1 unanswerable.
*Propositions: P1 (orderability), P4 (construct distinctness).*

## RQ3 — Domain invariance

**Does the structure remain invariant across domains?**

Tests whether the same 11-level structure instantiates in ≥3 unrelated domains
(agriculture, supply chain, clinical pathways) using only domain-specific *content*,
with **no levels added or removed**. Pre-registered null (**H0**): any domain
requiring a structural edit refutes invariance.
*Proposition: P2 (domain invariance — H1).*

## RQ4 — Predictive validity

**Does adaptive capability predict adaptation outcomes?**

Tests whether a system's measured level predicts real adaptation outcomes (shock
recovery, throughput/yield improvement per cycle) and **beats a domain-specific
baseline** maturity model with statistical significance. Turns the model from
taxonomy into predictive science.
*Proposition: P3 (predictive validity — H2).*

## RQ5 — Distinctness / incremental value

**Does adaptive capability provide explanatory power beyond existing maturity and
dynamic capability frameworks?**

The discriminant-validity guard. Tests whether the full ordered ladder explains
variance in adaptation outcomes that its nearest rivals — **CMMI** and **dynamic
capabilities** (Teece) — do not. Answers the committee's hardest challenge: "isn't
this just CMMI relabeled, or dynamic capabilities with extra steps?"
*Proposition: P5 (incremental validity).*

## Question → proposition → study map

| RQ | Question | Proposition | Hypothesis | Study |
|----|----------|-------------|------------|-------|
| RQ1 | Domain-invariant predictive construct? | — (conjunction of P2–P5) | — | All |

question is itself a publishable finding (see
